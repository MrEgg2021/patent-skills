# 专利发明点解析

从专利说明书全文中提取核心技术发明点，输出结构化 xlsx 表格。

## 核心原则

- **只吃说明书**：仅提取"技术领域"→"具体实施方式"之间的说明书文本，不喂摘要和权利要求（防止大模型偷懒、权重偏移）
- **不设篇幅上限**：技术问题和技术方案写充分，不截断、不过分精简
- **发明点数量**：模型自主判断有几个算几个，不设硬上限
- **严格原文**：所有内容必须来源于说明书，禁止编造

## 输入

| 格式 | 单篇 | 多篇 |
|------|------|------|
| .md / .txt | ✅ | ✅ |
| .pdf | ✅（自动调 pdf skill） | ✅ |
| .docx | ✅（自动调 docx skill） | ✅ |

## 输出

`.xlsx` 文件，一行一个专利，四列：

| 专利公开号 | 专利标题 | 技术问题 | 技术方案 |
|-----------|---------|---------|---------|

- **专利公开号**：从文件名或内容提取
- **专利标题**：说明书首行标题
- **技术问题**：多发明点时，序号 + 换行分段（如 `1. xxx\n2. xxx`）
- **技术方案**：与技术问题一一对应，同序号标注
- 列宽自适应，文本自动换行

## 工作流程

```
用户输入（文件/目录）
  → 识别格式（pdf → pdf skill, docx → docx skill）
  → 提取说明书全文（剔除摘要、权利要求）
  → 逐篇应用 invention_point_extraction_prompt.md
  → 保存 JSON
  → scripts/json_to_xlsx.py 生成 xlsx
```

## 提示词

使用 `references/invention_point_extraction_prompt.md`（基于 旦旦 推理版 V1.2），做以下调整：

1. **只喂说明书**：输入范围限定为说明书正文（技术领域→具体实施方式）
2. **输出格式改为表格友好**：单篇输出改为 `{"专利公开号": "...", "专利标题": "...", "技术问题": ["问题1", "问题2"], "技术方案": ["方案1", "方案2"]}`
   - 拆分逻辑：提示词原始输出中的 `"发明点"` 字段（格式为 `"技术问题：… 技术方案：…"`）需拆分为独立的 `技术问题` 和 `技术方案` 数组；`"文件名称"` 字段映射为 `专利公开号`；`专利标题` 从说明书首行提取。多发明点时数组各元素与序号对应。
3. **不设数量上限**：提示词中的"通常提炼一个最关键的发明点" → 改为"有几个提炼几个"
4. **不设篇幅约束**：去除"简洁明了""1-2句话"等篇幅限制

## 独立使用

直接给专利文件，输出 xlsx。不依赖其他 skill（PDF/Word 提取除外）。

JSON 转 xlsx 命令：

```bash
python3 scripts/json_to_xlsx.py invention_points.json 发明点解析.xlsx
```

脚本支持：
- LLM 输出的 ```json code block
- 单对象、对象数组、`{"results":[...]}`、`{"data":[...]}`
- `技术问题` / `技术方案` 为字符串或数组
- 旧字段：`文件名称`、`发明点`（含"技术问题/技术方案"文本）自动映射

## 作为 google-patent-search 预处理

被 Mode C 调用时：
```
专利文本 → invention-point-extraction → xlsx 中的技术问题+方案
  → 拼接为 technical_description_text → google-patent-search Mode A
```

## 工作目录

- 输出默认目录：当前工作目录，可通过 `OUTPUT_DIR` 环境变量自定义（`export OUTPUT_DIR=/path/to/output`）
