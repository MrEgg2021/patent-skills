# 专利申请初稿形式审查工具

> 基于 19 个知识产权保护中心预审自检表汇总的 **101 条审查规则**，对 DOCX 格式的中国发明/实用新型专利申请初稿进行自动化形式审查。

[English](./README_EN.md)

---

## ✨ 功能亮点

- **101 条全量规则**：覆盖说明书摘要（F-1~F-16）、权利要求书（F-17~F-46）、说明书（F-47~F-78）、说明书附图（F-79~F-101）四大部分
- **19 个保护中心覆盖**：吉林、徐州、浙江、北京、山西、甘肃、长沙、长春、福建、西安、内蒙古、陕西、安徽、辽宁、湖北、合肥、常州、苏州、天津
- **三类规则分级**：
  - 🔵 **A 类**（36 条）：纯脚本自动检查，无需人工介入
  - 🟡 **B 类**（42 条）：脚本提取 + LLM 判断，兼顾效率与准确性
  - 🔴 **C 类**（23 条）：纯 LLM 语义审查，处理需理解的复杂规则
- **中心选择机制**：选择目标保护中心后，101 条全量审核，但分歧项/独有项按所选中心标准判定，未覆盖中心启用最严格模式
- **结构化 DOCX 报告**：8 章节专业审查报告，含问题定位、定性、来源中心、修改建议

---

## 📋 审查范围

| 审查部分 | 规则编号 | 规则数量 |
|---------|---------|---------|
| 说明书摘要 | F-1 ~ F-16 | 16 |
| 权利要求书 | F-17 ~ F-46 | 30 |
| 说明书 | F-47 ~ F-78 | 32 |
| 说明书附图 | F-79 ~ F-101 | 23 |

**仅支持**：中国发明专利、实用新型专利的 DOCX 格式申请文件初稿

**不在范围内**：外观设计专利、请求书、委托书、意见陈述书

## 🚀 快速开始

### 依赖安装

```bash
pip install python-docx Pillow lxml
```

### 使用方式

本工具设计为 [WorkBuddy](https://www.codebuddy.cn) Skill 运行，也可独立使用核心脚本：

```python
from scripts.docx_parser import DocxSectionParser
from scripts.formality_scanner import FormalityScanner

# 1. 解析 DOCX 文件
parser = DocxSectionParser("你的专利文件.docx")
parsed_doc = parser.parse()

# 2. 执行 A 类自动扫描（指定目标保护中心）
scanner = FormalityScanner(parsed_doc, target_center="安徽")
results = scanner.run_all()

# 3. 查看结果
for r in results:
    print(f"[{r['rule_id']}] {r['status']} - {r['message']}")
```

在 WorkBuddy 中，直接输入触发词即可启动完整审查流程：

> 形式审查 / 形式自检 / 预审自检 / 预审检查 / 格式扫描 / 格式检查

## 🧠 核心设计

### 三类规则体系

| 类型 | 数量 | 执行方式 | 典型规则 |
|------|------|---------|---------|
| A 类（自动） | 36 条 | 纯 Python 脚本检查 | 摘要字数、附图标记格式、权利要求编号连续性 |
| B 类（混合） | 42 条 | 脚本提取内容 + LLM 判断 | 权利要求引用关系、说明书附图标记一致性 |
| C 类（语义） | 23 条 | 纯 LLM 语义审查 | 技术方案完整性、附图说明充分性 |

### 中心选择三种判定场景

| 场景 | 示例 | 判定方式 |
|------|------|---------|
| 所选中心在**严格方** | 选安徽 → 附图彩色 | 按严格标准判定 🔴 |
| 所选中心在**宽松方** | 选吉林 → 附图彩色 | 按宽松标准判定 ✅，附注其他中心要求 |
| 所选中心**未提及** | 选浙江 → 附图彩色 | 标记为 💡建议关注 |

## 📊 审查报告结构

生成的 DOCX 审查报告包含 8 个章节：

```
1. 审查概况       → 文件信息、专利类型、规则统计、各级别问题数量
2. 说明书摘要审查  → F-1~F-16 自动扫描 + LLM 审查结果
3. 权利要求书审查  → F-17~F-46 自动扫描 + LLM 审查结果
4. 说明书审查     → F-47~F-78 自动扫描 + LLM 审查结果
5. 说明书附图审查  → F-79~F-101 自动扫描 + LLM 审查结果
6. 建议关注项     → 非所选中心硬性要求但其他中心有此要求的检查结果
7. 人工确认事项   → 无法自动判断的检查项
8. 分歧项说明     → 各中心要求差异及所选中心立场
```

**问题定性等级**：

- 🔴 **必须修改** — 不符合硬性要求，必须修改后才能提交
- 🟡 **建议优化** — 非强制但影响审查通过率
- ⚪ **需人工确认** — 脚本/LLM 无法确定，需人工复核
- 💡 **建议关注** — 非所选中心硬性要求，但其他中心有此要求
