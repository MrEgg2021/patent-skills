# 权利要求技术特征拆解

## 功能

将专利文件的权利要求按语义单位拆解为独立技术特征，并从说明书中智能匹配每个特征的详细描述。输出三列 xlsx。

## 适用场景

- 专利对比分析的前置步骤（为 [[patent-comparison-report]] 提供输入）
- OA 答复中逐特征比对
- 侵权分析中特征划分

## 工作流程

```
专利文件 → 提取权利要求+说明书 → AI 推理（推理模型版提示词）→ JSON → json_to_xlsx.py → xlsx
```

## 用法

### 步骤 1：提取专利文本

从专利文件中提取权利要求书和说明书全文。支持 .pdf / .docx / .md / .txt。

### 步骤 2：调用 AI 推理

将权利要求书和说明书文本，连同以下提示词一起发送给 AI：

**提示词文件**：`references/claims_feature_decomposition_reasoning.md`

向 AI 提供：
1. 权利要求书全文
2. 说明书全文（技术领域→具体实施方式，不含摘要）
3. 提示词内容

AI 将输出一个 JSON 数组，格式如下：

```json
[
  {
    "权利要求序号": "1",
    "技术特征 (源自权利要求书原文，按语义单位独立分解)": "原文",
    "说明书描述 (主题归类式整合，严格源自说明书原文，并注明来源)": "描述"
  }
]
```

### 步骤 3：转换 xlsx

```bash
python3 scripts/json_to_xlsx.py output.json 权利要求特征拆解.xlsx
```

## 输出格式

| 列 | 内容 |
|----|------|
| 权利要求序号 | 含引用关系，如 `2 (引1)` |
| 技术特征 | 权利要求原文，按语义单位独立分解 |
| 说明书描述 | 主题归类整合（工作原理/实施例/参数/效果等），附来源段落号 |

## 技术领域

`domain: "全领域"` — 适用于电学、机械、化学、生物等所有技术领域。

## 环境

- Python ≥ 3.9
- 依赖：`pandas`、`openpyxl`
- 安装：`pip install pandas openpyxl`
