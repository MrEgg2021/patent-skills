# 技术交底书审核清单

共 17 条规则，分 6 个模块。源自专利初稿审核的 37 条规则中可逆推至交底书阶段的子集。

---

## D.1 方案完整性（3 条）

### Rule D.1.1 必要特征完整性 🔴

检查交底书是否交代了解决技术问题所必需的全部结构部件或方法步骤。是否存在"一个××装置，包括 A、B……"但 B 的工作方式未描述的情况？

> 转化自：Rule 1.8 必要技术特征检查

### Rule D.1.2 方法步骤闭环 🔴

方法类交底书是否描述了从开始到结束的完整步骤序列？是否存在"然后进行信号处理"等跳步式描述？

> 转化自：Rule 1.12 方法步骤时序检查

### Rule D.1.3 结构连接关系 🔴

结构类交底书是否交代了各部件之间的位置关系和连接方式（固定/可拆卸/活动）？是否存在"A 连接于 B"但未说明如何连接的情况？

> 转化自：Rule 1.13 位置/连接关系检查

---

## D.2 逻辑链（2 条）

### Rule D.2.1 问题-方案-效果闭环 🔴

交底书是否形成了"现有技术的具体缺陷 → 本方案如何解决该缺陷 → 解决后产生了什么技术效果"的完整逻辑链？是否存在"技术问题"其实是"技术效果"的重述（如"本发明的目的是提高效率"，但未说明现有方案的效率瓶颈在哪）？

> 转化自：Rule 2.3 发明内容对应性检查

### Rule D.2.2 技术问题事实支撑 🔵

交底书是否给出了足够事实支撑来说明该技术问题确实存在？是否说明了现有方案的具体缺陷、限制或不足？不得凭外部常识直接断言问题真实或虚假；材料不足时，应要求发明人补充背景事实、使用场景、测试现象或对比说明。

---

## D.3 可实施性（3 条）

### Rule D.3.1 术语定义 🔴

交底书中出现的核心术语（尤其是发明人自创的术语、领域缩写、专有名词），是否在首次出现时给出了清晰的定义？是否存在全文使用了一个关键术语但从未解释其含义的情况？

> 转化自：Rule 2.8 术语定义检查

### Rule D.3.2 公式可理解性 🔴

交底书中的公式是否附有自然语言解释？公式中的每个变量是否定义了物理含义和取值范围？是否存在只给出公式不解释的情况？

> 转化自：Rule 1.10 公式使用检查

### Rule D.3.3 实施例具体性 🔴

交底书是否提供了至少一个具体的、可操作的实施例？实施例是否足够详细，使得本领域技术人员能够复现？是否存在"通过优化算法得到最优解"这类不交代具体手段的描述？

> 转化自：Rule 2.9 上位概念支持度检查 + Rule 2.10 数值范围支持度检查

---

## D.4 附图与表述（3 条）

### Rule D.4.1 附图完整性 🔴

交底书是否包含附图？如含，附图是否足以帮助理解技术方案？是否存在"参见图 3"但交底书内无图 3 的情况？

> 转化自：Rule 2.11 附图文字一致性检查

### Rule D.4.2 图文对应性 🔴

交底书文字描述中提到的部件/步骤，是否在附图中均有对应标注？附图中的标号是否与文字描述一致？

> 转化自：Rule 2.11 附图文字一致性检查

### Rule D.4.3 术语一致性 🔵

同一概念在交底书全文中是否使用了统一的术语？是否存在同一部件在不同段落有不同名称（如前面叫"固定座"后面叫"安装座"）？

---

## D.5 创造性支撑线索（3 条）

### Rule D.5.1 与现有技术的区别线索 🔴

交底书是否明确说明了本方案与现有方案、常规做法或发明人已知方案的区别？是否描述了区别点带来的技术效果或技术贡献？如果没有现有技术描述，不得直接推断新颖性/创造性结论，只能要求发明人补充区别点、已有方案和效果依据。

### Rule D.5.2 数值/参数依据 🔵

交底书中出现的数值（阈值、范围、比例等），是否有来源说明？是实验数据、经验值、理论推导还是行业标准？是否存在无依据的"优选 0.5-0.8"？

> 转化自：Rule 2.10 数值范围支持度检查

### Rule D.5.3 算法/AI 方案完整性 🔴（电学/AI 特化）

对于涉及算法或 AI 模型的交底书：
- 是否描述了算法的输入、处理逻辑和输出？
- 是否描述了模型的基本结构（如网络层级、模块构成）？
- 是否描述了训练方法或数据处理方式的基本特征？

> 转化自：Rule 2.12 AI充分公开检查

> 与 `domain_electrical.md` 中 E.1/E.2 存在重叠时，合并输出为一条问题，并同时标注涉及规则编号。

---

## D.6 自检（3 条）

### Rule D.6.1 自限性表述 🔵

交底书中是否存在"必需的""关键的""最重要的""唯一的"等在未来审查中可能构成不当限制的绝对化用语？是否存在"填补国内空白"等商业宣传用语？

> 转化自：Rule 2.4 禁用词检查

### Rule D.6.2 代码可读性 🔵（软件特化）

如果交底书包含代码片段，是否配有自然语言或伪代码解释？是否存在大段源代码但没有文字说明的情况？

> 转化自：Rule 2.14 代码处理检查

### Rule D.6.3 逻辑矛盾 🔴

交底书全文中是否存在前后矛盾的情况？例如：前面说"A 是固定的"，后面说"A 可活动"；或附图与文字描述不一致。
